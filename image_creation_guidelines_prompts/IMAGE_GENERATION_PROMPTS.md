# Image Generation — Full Exact Prompts

All prompts, templates, and API payloads used to generate images in the Vision GridAI pipeline. Captured verbatim from the live n8n workflows and seed templates.

---

## 1. Final Prompt Sent to Fal.ai Seedream 4.5

Each scene's final prompt is assembled in `WF_IMAGE_GENERATION.json` → **Fire All Scenes** node:

```js
// Build 3-part cinematic prompt
const prefix   = scene.composition_prefix || '';
const subject  = scene.image_prompt || '';
const parts    = [prefix, subject, styleDna].filter(Boolean);
const fullPrompt = parts.join(' ');
```

### Formula

```
{COMPOSITION_PREFIX} {SCENE_SUBJECT} {STYLE_DNA}
```

- `COMPOSITION_PREFIX` — one of 8 values (table below) stored per scene in `scenes.composition_prefix`
- `SCENE_SUBJECT` — the `image_prompt` generated by Claude during script metadata pass (template below)
- `STYLE_DNA` — locked per project, stored in `projects.style_dna`, appended to EVERY scene

## 2. Universal Negative Prompt

Hardcoded in `WF_IMAGE_GENERATION.json` (Fire All Scenes node) and sent on every Fal.ai call as `negative_prompt`:

```
text, watermark, signature, logo, UI elements, blurry, low quality, distorted faces, extra fingers, mutated hands, bad anatomy, deformed, disfigured, out of frame, cropped, duplicate, error, jpeg artifacts, low resolution, cartoon, anime, painting, illustration, 3D render, CGI
```

## 3. Composition Prefix Library (8 options)

Stored in `prompt_templates` table. Claude selects one per scene during script generation.

| composition_prefix   | Exact Prompt Prefix                                                                  | Best For                                 |
|----------------------|--------------------------------------------------------------------------------------|------------------------------------------|
| `wide_establishing`  | `wide angle establishing shot, full scene visible, environmental context,`          | Opening hooks, location reveals          |
| `medium_closeup`     | `medium close-up shot, subject fills center frame, shoulders and above,`            | Character focus, emotional beats         |
| `over_shoulder`      | `over the shoulder perspective, looking outward from behind subject,`               | Immersive moments                        |
| `extreme_closeup`    | `extreme close-up, tight crop on detail, shallow depth of field,`                   | B-roll, product shots, detail emphasis   |
| `high_angle`         | `high angle shot looking down, bird's eye perspective, subject below,`              | Power dynamics, vulnerability            |
| `low_angle`          | `low angle shot looking up, subject towering above, dramatic perspective,`          | Authority, grandeur                      |
| `symmetrical`        | `perfectly symmetrical composition, centered subject, architectural framing,`       | Formal scenes, dramatic reveals          |
| `leading_lines`      | `composition using strong leading lines converging to subject,`                     | Pathways, corridors, perspective depth   |

## 4. Style DNA Templates (4 base templates, LOCKED per project)

Generated once during project creation, stored in `projects.style_dna`, appended to EVERY image prompt.

### Historical / Period Drama
```
cinematic still frame, {ERA} period accuracy, dramatic chiaroscuro lighting, 35mm film grain texture, desaturated muted color palette, deep shadows, documentary photorealism, volumetric fog, depth of field bokeh, 8K detail
```

### Modern Finance / Premium
```
modern commercial photography, clean minimalist composition, premium luxury aesthetic, soft directional studio lighting, subtle depth of field, neutral background tones, crisp 4K sharpness, contemporary color palette, professional product photography feel
```

### Tech / Futuristic
```
futuristic cinematic still, cool blue-purple ambient lighting, clean glass and steel surfaces, subtle neon accents, high contrast shadows, modern tech aesthetic, volumetric light rays, ultra-sharp detail, dark background with rim lighting
```

### Warm Human / Inspirational
```
warm golden hour photography, natural soft lighting, rich earth tones, gentle lens flare, human-centered composition, emotional depth, 35mm film warmth, intimate documentary feel, subtle grain, shallow focus on subject
```

## 5. Claude Metadata Prompt — Generates Per-Scene `image_prompt`

Sent to Anthropic Claude in `WF_SCRIPT_GENERATE.json` (Prepare Metadata Prompt node) after mechanical sentence-split into ~110-word chunks. Claude returns only metadata (image_prompt, emotional_beat, chapter) — never touches narration.

### Exact prompt (with `' + chunks.length + '` and `' + chunkList` as runtime interpolations)

```
Below are {N} text chunks from a documentary script about "{SEO_TITLE}".

For EACH chunk, generate ONLY:
1. image_prompt: 40-80 words, cinematic. Structure: [SUBJECT] + [ENVIRONMENT] + [LIGHTING] + [COMPOSITION] + [MOOD] + [DETAILS]. Be hyper-specific. People as silhouettes/hands/figures from behind. Include lighting direction. No text/words/signs in images.
2. emotional_beat: one of hook, tension, revelation, data, story, resolution, transition
3. chapter: a short chapter name grouping related scenes

Return ONLY a JSON array: [{"scene_number": 1, "image_prompt": "...", "emotional_beat": "...", "chapter": "..."}]

Do NOT return or modify any narration text. I only need image_prompt, emotional_beat, and chapter for each chunk number.

CHUNK 1: {chunk_1_text}

CHUNK 2: {chunk_2_text}

...
```

### JS code that builds it (`WF_SCRIPT_GENERATE.json` → Prepare Metadata Prompt)

```js
const chunkList = chunks.map((c, i) => 'CHUNK ' + (i+1) + ': ' + c).join('\n\n');

const metadataPrompt =
  'Below are ' + chunks.length + ' text chunks from a documentary script about "' +
  (topic.seo_title || topic.original_title) + '".\n\n' +
  'For EACH chunk, generate ONLY:\n' +
  '1. image_prompt: 40-80 words, cinematic. Structure: [SUBJECT] + [ENVIRONMENT] + [LIGHTING] + [COMPOSITION] + [MOOD] + [DETAILS]. Be hyper-specific. People as silhouettes/hands/figures from behind. Include lighting direction. No text/words/signs in images.\n' +
  '2. emotional_beat: one of hook, tension, revelation, data, story, resolution, transition\n' +
  '3. chapter: a short chapter name grouping related scenes\n\n' +
  'Return ONLY a JSON array: [{"scene_number": 1, "image_prompt": "...", "emotional_beat": "...", "chapter": "..."}]\n\n' +
  'Do NOT return or modify any narration text. I only need image_prompt, emotional_beat, and chapter for each chunk number.\n\n' +
  chunkList;
```

## 6. Fal.ai API Payload

Scene processor (`WF_SCENE_IMAGE_PROCESSOR.json`) sends this exact body to Fal.ai Seedream 4.5:

### Endpoint
```
POST https://queue.fal.run/fal-ai/bytedance/seedream/v4.5/text-to-image
Authorization: Key {{FAL_API_KEY}}
Content-Type: application/json
```

### Body (n8n expression)
```
={{ JSON.stringify({ prompt: $json.image_prompt, negative_prompt: $json.negative_prompt || undefined, image_size: 'landscape_16_9' }) }}
```

### Body (resolved at runtime)
```json
{
  "prompt": "{COMPOSITION_PREFIX} {SCENE_SUBJECT} {STYLE_DNA}",
  "negative_prompt": "text, watermark, signature, logo, UI elements, blurry, low quality, distorted faces, extra fingers, mutated hands, bad anatomy, deformed, disfigured, out of frame, cropped, duplicate, error, jpeg artifacts, low resolution, cartoon, anime, painting, illustration, 3D render, CGI",
  "image_size": "landscape_16_9"
}
```

- `image_size: "landscape_16_9"` → long-form (1920×1080)
- `image_size: "portrait_9_16"` → shorts (1080×1920)
