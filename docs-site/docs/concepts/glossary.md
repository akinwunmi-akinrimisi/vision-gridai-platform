# Glossary

Operator-facing terms used across the platform, dashboard, n8n workflows, and
Supabase schema.

**Approval Gate**
:   One of four pause points in the pipeline (Topics → Script → Video →
    Shorts). Each gate writes its decision to a single status column on
    `topics` or `shorts`. See [the 4 approval gates](gates.md).

**Auto-Pilot Mode**
:   Per-project setting that lets the pipeline skip Gates 1, 2, and 3 when
    score thresholds are met. Always publishes as `unlisted` — never public.
    Auto-pauses if monthly spend exceeds budget or two consecutive videos
    score below 6.0. Source: `CLAUDE.md` "Auto-pilot mode per project".

**Caption Burn Service**
:   Host-side Python systemd service at `/opt/caption-burn/caption_burn_service.py`
    listening on port `:9998`. Runs FFmpeg + libass via `docker exec
    n8n-n8n-1 ffmpeg ...` to bypass the n8n task runner OOM that occurs when
    re-encoding inside the container. Source:
    `directives/06-captions-assembly.md:53-54,70`.

**Composition Prefix**
:   One of 8 image-prompt openers (`wide_establishing`, `medium_closeup`,
    `over_shoulder`, `extreme_closeup`, `high_angle`, `low_angle`,
    `symmetrical`, `leading_lines`) prepended to every Seedream image prompt
    along with the project's Style DNA. Stored in `scenes.composition_prefix`,
    seeded in the `prompt_templates` table. Source:
    `directives/02-script-generation.md:44`.

**Cost Calculator**
:   Pipeline gate at Stage 7.5 (between scene segmentation and TTS) that asks
    the operator to choose the I2V video ratio: 100/0, 95/5, 90/10, or 85/15.
    Determines the per-video cost between $13.76 and $76.65. See
    [pipeline economics](economics.md#the-cost-calculator-gate-stage-75).

**Force Pass**
:   When the script evaluator fails to clear the 7.0 combined-evaluation
    threshold across 3 retries, the script is "force-passed" and surfaced to
    the operator with a warning banner. `topics.script_force_passed = true`.
    Force-passed scripts always require human review at Gate 2 even with
    auto-pilot enabled. Source:
    `directives/02-script-generation.md:82` + `dashboard/src/components/script/ForcePassBanner.jsx`.

**Hook Score**
:   Per-chapter retention score (1-10) generated by CF12 (`WF_HOOK_ANALYZER`)
    during Pass 3 of script generation. Stored in `topics.viral_moments`.
    Surfaces in Gate 2 dashboard alongside the 7-dimension quality score.
    Source: `directives/10-intelligence-layer.md:31`.

**Hybrid Scene**
:   Scene assembled from Ken Burns clip + crossfade + I2V clip + crossfade +
    Ken Burns when the scene is upgraded to video by the Cost Calculator
    selection. Seedance audio is discarded; TTS remains the audio source.
    Source: `CLAUDE.md` "Hybrid scene assembly" gotcha.

**Ken Burns**
:   FFmpeg `zoompan` motion applied to every static Seedream image, producing
    a .mp4 clip whose duration equals the scene's TTS audio duration. Six
    direction templates (`zoom_in_center`, `zoom_out_center`, `pan_left`,
    `pan_right`, `pan_up`, `pan_down`). Zoom intensity 0.001 for long-form,
    0.0015 for shorts. Source: `directives/05-ken-burns-color.md:19-29`.

**Lyria**
:   Google Vertex AI model (`lyria-002`) that generates custom 30-second
    background-music clips per chapter. Output is mixed under voiceover at
    `volume=0.12` via FFmpeg `amix` ducking. Source: `CLAUDE.md` "Background
    Music (Lyria)" + `GUIDE.md` Phase 3.

**Master Clock**
:   The TTS audio is the single source of truth for all scene durations.
    FFprobe measures actual duration (never estimated) and writes it to
    `scenes.audio_duration_ms`. Every visual derives its length from this
    field. Source: `CLAUDE.md` "Audio is the master clock".

**Outlier Score**
:   0-100 score from CF01 (`WF_OUTLIER_SCORE`) measuring algorithmic-push
    potential. Combines YouTube search volume, competing video count, and
    freshness signals. Stored in `topics.outlier_score` and
    `topics.algorithm_momentum`. Source:
    `directives/01-topic-generation.md:21,34`.

**PPS (Pipeline Performance Score)**
:   Predictive performance score from CF13 (`WF_PREDICT_PERFORMANCE`)
    calculated after assembly. Stored in `topics.predicted_performance_score`.
    Calibration job (`WF_PPS_CALIBRATE`) runs monthly to reweight the model
    against actual analytics. Source: `directives/10-intelligence-layer.md:32,68,90`.

**Production Register**
:   One of 5 visual-style registers (`REGISTER_01_ECONOMIST`,
    `REGISTER_02_PREMIUM`, `REGISTER_03_NOIR`, `REGISTER_04_SIGNAL`,
    `REGISTER_05_ARCHIVE`) selected per scene by the scene classifier.
    Determines which Style DNA variant and negative-prompt additions are
    applied. Stored in `scenes.production_register`. Source:
    `supabase/migrations/023_production_register.sql:21-22` +
    `image_creation_guidelines_prompts/REGISTER_01_IMAGE_PROMPTS.md`.

**Resume/Checkpoint**
:   Pattern in every production workflow: before processing a scene, query
    its status column (`audio_status`, `image_status`, `clip_status`) and
    skip if not `pending`. Lets a crashed workflow restart from where it
    left off. `topics.pipeline_stage` tracks last completed stage globally.
    Source: `CLAUDE.md` "Resume/checkpoint on every production workflow".

**RPM (Revenue Per Mille)**
:   Estimated revenue per 1,000 YouTube views, classified per niche by
    `WF_RPM_CLASSIFY` (CF03) at project creation. Stored in
    `projects.niche_rpm_category` and `estimated_rpm_low/mid/high`. Used by
    the cost-vs-revenue analytics. Source:
    `directives/00-project-creation.md:24,32`.

**Seedance**
:   Fal.ai I2V model (`fal-ai/bytedance/seedance/2.0/fast`) at $0.2419 per
    second for 720p 10-second clips. Used only for the percentage of scenes
    selected at the Cost Calculator gate. Output is upscaled to 1080p in
    FFmpeg via `scale=1920:1080:flags=lanczos`. Source: `CLAUDE.md` Tech
    Stack + "Seedance 2.0 Fast outputs 720p" gotcha.

**Seedream**
:   Fal.ai image model (`fal-ai/bytedance/seedream/v4.5/text-to-image`) at
    $0.04 per image. Generates the static still for every scene
    (long-form 16:9, shorts 9:16). Source: `CLAUDE.md` Tech Stack.

**Selective Color**
:   Scene-level effect that converts the image to monochrome except for one
    named element (e.g., "the red umbrella"). When `scenes.selective_color_element
    IS NOT NULL`, the FFmpeg color-grade filter chain is skipped entirely
    and the prompt injects the selective-color instruction. Source:
    `directives/05-ken-burns-color.md:60` + `CLAUDE.md` "Selective color
    scenes SKIP FFmpeg color grading" gotcha.

**Self-chaining**
:   Architecture pattern where each n8n workflow fires the next workflow
    on completion via webhook. `WF_MASTER` is only a launcher. Every
    workflow handles its own errors and writes failure to Supabase.
    Source: `CLAUDE.md` "Self-chaining architecture".

**SEO Score**
:   0-100 score from CF02 (`WF_SEO_SCORE`) measuring search-pull potential.
    Combines keyword difficulty, search volume, and content-gap analysis.
    Stored in `topics.seo_score`, `seo_classification`, `primary_keyword`.
    Source: `directives/01-topic-generation.md:22,35`.

**Style DNA**
:   Locked visual identity per project. Generated during project creation,
    stored in `projects.style_dna`, appended to **every** image prompt as
    `composition_prefix + scene_subject + style_dna`. Never modified between
    scenes. Five register-specific variants exist in `prompt_templates`.
    Source: `image_creation_guidelines_prompts/REGISTER_01_IMAGE_PROMPTS.md:21-47`
    + `CLAUDE.md` "Style DNA is LOCKED per project".

**Topic Intelligence**
:   On-demand 5-source research engine at the global `/research` route
    (Reddit, YouTube Comments, TikTok, Google Trends + PAA, Quora). Mines
    keywords derived dynamically from a project's niche/description and
    feeds candidate topics into the project-creation modal. Cost ~$0.13/run.
    Source: `CLAUDE.md` "Topic Intelligence research engine" +
    `directives/topic-intelligence/`.

**Universal Negative Prompt**
:   Shared negative prompt sent on every Fal.ai image call to suppress
    artifacts, text in images, and style drift. Stored once in
    `prompt_templates` (template_type = `negative`); register-specific
    additions are concatenated at request time. Source: `CLAUDE.md`
    "Universal negative prompt on ALL Fal.ai calls" +
    `image_creation_guidelines_prompts/REGISTER_01_IMAGE_PROMPTS.md:51-58`.
