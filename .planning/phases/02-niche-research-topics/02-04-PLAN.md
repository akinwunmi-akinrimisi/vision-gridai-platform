**WF_TOPICS_GENERATE.json** -- n8n workflow for generating 25 topics + 25 avatars:

1. **Webhook: POST /webhook/topics/generate** -- Receives: { project_id }

2. **HTTP Request: Read project + prompt_configs** -- GET projects?id=eq.{{project_id}} to get niche, playlists, expertise. GET prompt_configs?project_id=eq.{{project_id}}&prompt_type=eq.topic_generator&is_active=eq.true to get the topic generation prompt.

3. **Code: Build prompt** -- Inject niche, playlist names/themes, expertise profile, blue-ocean strategy into the topic_generator prompt template. Add instruction to generate 25 topics (8-9 per playlist) with full avatar (10 data points each). Output format: JSON array with topic objects and nested avatar objects.

4. **HTTP Request: Update project status** -- PATCH projects status to 'generating_topics'.

5. **Respond to Webhook** -- Return { success: true } immediately (async pattern).

6. **HTTP Request: Anthropic API** -- POST to api.anthropic.com/v1/messages with the built prompt. Model: claude-sonnet-4-6, max_tokens: 16384. No web_search tool needed for topic generation. Timeout: 120s.

7. **Code: Parse topics** -- Extract JSON from response content blocks. Parse into array of 25 topic objects. Validate each has: topic_number, playlist_group, playlist_angle, seo_title, narrative_hook, key_segments, estimated_cpm, viral_potential, plus nested avatar with 10 fields.

8. **Code: Build Supabase INSERT arrays** -- Create two arrays:
   - topics_array: 25 rows with project_id, all topic fields, review_status='pending', status='pending'
   - avatars need topic_ids, so topics must be inserted first to get IDs back

9. **HTTP Request: INSERT topics** -- POST to topics table with Prefer: return=representation to get back generated UUIDs.

10. **Code: Map topic IDs to avatars** -- Match each avatar to its topic by topic_number, set topic_id and project_id.

11. **HTTP Request: INSERT avatars** -- POST avatars array to avatars table.

12. **HTTP Request: UPDATE project** -- PATCH project status to 'topics_pending_review'.

Error handling: On any failure, PATCH project with status='topic_generation_failed' and error_log.

**WF_TOPICS_ACTION.json** -- n8n workflow for topic approve/reject/refine/edit:

1. **Webhook: POST /webhook/topics/action** -- Receives: { action, topic_ids, feedback, instructions, fields, project_id }. Action is one of: 'approve', 'reject', 'refine', 'edit', 'regenerate-rejected'.

2. **Code: Route by action** -- Switch/case on action field.

3. **approve branch**: PATCH topics SET review_status='approved' WHERE id IN (topic_ids). Respond: { success: true }.

4. **reject branch**: PATCH topics SET review_status='rejected', review_feedback=feedback WHERE id IN (topic_ids). Respond: { success: true }.

5. **refine branch** (most complex):
   - PATCH topic SET review_status='refining'
   - GET all 25 topics for this project (to pass 24 others as context per CLAUDE.md requirement)
   - GET prompt_configs for topic_generator prompt
   - Build Claude prompt: "Refine this topic based on the following instructions. Here are the other 24 topics for context -- avoid overlap. Instructions: {{instructions}}. Topic to refine: {{topic_json}}. Other topics: {{other_topics_json}}."
   - HTTP Request: Anthropic API with the refine prompt
   - Parse response, extract refined topic fields
   - PATCH topic with new fields + append to refinement_history JSONB array: { instruction, result, timestamp }
   - PATCH topic SET review_status='pending' (back to pending after refine)
   - Respond: { success: true }

6. **edit branch**: PATCH topic SET fields from request body. Also PATCH avatar if avatar fields included. Respond: { success: true }.

7. **regenerate-rejected branch**:
   - GET all topics WHERE project_id=eq.X AND review_status=eq.rejected
   - GET all topics WHERE project_id=eq.X (for context)
   - For each rejected topic: call Claude to regenerate with context of all other topics + original rejection feedback
   - PATCH each regenerated topic + avatar
   - SET review_status='pending'
   - Respond: { success: true, regenerated_count: N }

All Supabase calls use apikey + Authorization headers with n8n credential references. All Anthropic calls use httpHeaderAuth credential.

**IMPORTANT:** Both files must be valid n8n workflow JSON importable via n8n UI.
