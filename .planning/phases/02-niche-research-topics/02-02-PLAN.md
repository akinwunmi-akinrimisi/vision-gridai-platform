  <action>
Create an n8n workflow JSON file that implements the full project creation pipeline. This workflow is the backend for the /webhook/project/create endpoint.

**Workflow structure (12 nodes):**

1. **Webhook node** -- POST /webhook/project/create. Receives: { niche, description, target_video_count }

2. **Code node: Validate Input** -- Check niche is non-empty string. Set defaults: target_video_count=25 if missing, channel_style='2hr_documentary'.

3. **HTTP Request: INSERT project** -- POST to supabase.operscale.cloud/rest/v1/projects. Body: { name: niche, niche, niche_description: description, target_video_count, status: 'created' }. Headers: apikey, Authorization Bearer SERVICE_ROLE_KEY, Prefer: return=representation. Extract project id from response.

4. **Respond to Webhook** -- Return immediately: { success: true, data: { id: project_id } }. This is the ASYNC pattern -- dashboard gets ID immediately, watches Realtime for progress.

5. **HTTP Request: Update status** -- PATCH projects?id=eq.{{project_id}}, body: { status: 'researching_competitors' }

6. **HTTP Request: Anthropic API (niche research)** -- POST to https://api.anthropic.com/v1/messages. Headers: x-api-key from n8n credential (httpHeaderAuth), anthropic-version: 2023-06-01, content-type: application/json. Body:
```json
{
  "model": "claude-sonnet-4-6",
  "max_tokens": 16384,
  "tools": [{ "type": "web_search_20250305", "name": "web_search", "max_uses": 10 }],
  "messages": [{ "role": "user", "content": "Research the niche '{{niche}}' for a YouTube documentary channel. Using web search:\n\n1. COMPETITOR AUDIT: Find top 10 YouTube channels in this niche. For each: channel name, subscriber count, avg views, topics they cover (RED OCEAN), gaps they miss.\n\n2. AUDIENCE PAIN POINTS: Search Reddit, Quora, and forums for: most upvoted questions, common frustrations, what people say is missing from YouTube coverage.\n\n3. KEYWORD GAPS: Find high-volume low-competition keywords.\n\n4. BLUE OCEAN ANALYSIS: Identify 5-8 content angles no top channel covers well. Value curve gaps. Channel positioning statement.\n\n5. NICHE PROFILE: Generate a dynamic system prompt (what expertise AI should embody), expertise description, 3 playlist angles with names and themes, 10+ red-ocean topics to AVOID, primary audience psychographic.\n\nReturn as JSON with keys: competitor_analysis, audience_pain_points, keyword_research, blue_ocean_opportunities, expertise_profile, system_prompt, playlist_angles (array of 3 with name+theme), red_ocean_topics (array), audience_psychographic." }]
}
```
Set timeout to 120 seconds (Claude + web search can take 60+ seconds).

7. **Code node: Parse research response** -- Extract text content from response: `response.content.filter(c => c.type === 'text').map(c => c.text).join('')`. Parse the JSON from the text. Handle case where Claude wraps JSON in markdown code fences. Extract each field into separate variables.

8. **HTTP Request: INSERT niche_profiles** -- POST to niche_profiles table with competitor_analysis, audience_pain_points, keyword_research, blue_ocean_opportunities as JSONB.

9. **HTTP Request: UPDATE projects** -- PATCH projects with playlist1_name/theme, playlist2_name/theme, playlist3_name/theme, niche_system_prompt, niche_expertise_profile, niche_red_ocean_topics, niche_blue_ocean_strategy, status: 'researching_prompts'.

10. **HTTP Request: Anthropic API (prompt generation)** -- Second Claude call to generate dynamic prompts. Sends the niche profile + expertise as context. Asks Claude to generate: topic_generator prompt, script_pass1/2/3 prompts, evaluator prompt, visual_director prompt. Each as a complete prompt template with {{variable}} placeholders.

**CRITICAL for TOPC-02 compliance:** The topic_generator prompt MUST include these quality benchmarks explicitly in its instructions:
- **2 AM Test**: "Would someone search for this topic at 2 AM when they can't sleep?"
- **Share Test**: "Would someone share this video with a friend unprompted?"
- **Rewatch Test**: "Would someone rewatch this or bookmark it for later?"
Each generated topic must be evaluated against all three tests. The prompt must instruct Claude to score each topic on these dimensions and only include topics that pass at least 2 of 3 tests.

11. **Code node: Parse prompts + build INSERT array** -- Parse the prompt response. Build array of 7 prompt_configs rows: [{ project_id, prompt_type: 'topic_generator', prompt_text, version: 1, is_active: true }, ...].

**CRITICAL validation in this Code node:** After parsing the topic_generator prompt, verify it contains the quality benchmark keywords. Check: `if (!topicGeneratorPrompt.includes('2 AM') || !topicGeneratorPrompt.includes('Share Test') || !topicGeneratorPrompt.includes('Rewatch')) { throw new Error('topic_generator prompt missing quality benchmarks'); }`. This ensures TOPC-02 compliance at the workflow level.

12. **HTTP Request: INSERT prompt_configs** -- POST to prompt_configs (bulk insert via Supabase REST API -- send array). Then PATCH projects status to 'ready_for_topics'.

**Error handling:** Each HTTP Request node should have continueOnFail=true. After the Anthropic calls, add a Code node that checks for errors and writes { status: 'research_failed', error_log: error_message } to the projects table if anything fails.

**Credential references:** Use n8n credential references (not hardcoded keys):
- Supabase: httpHeaderAuth credential with apikey + Authorization
- Anthropic: httpHeaderAuth credential with x-api-key

**IMPORTANT:** This creates a workflow JSON file for import into n8n. The file should be valid n8n workflow JSON that can be imported via n8n UI (Settings > Import Workflow).
