COMMON FIX PATTERNS (apply only if the corresponding bug was found):

Pattern A: WF_SCRIPT_GENERATE timeout (topics stuck at status=scripting)
- Cause: A Claude HTTP Request node timed out and the error wasn't caught
- Fix: Ensure all Claude nodes have onError: continueRegularOutput AND the error check Code node runs after each one
- Specific: If Pass 2 timed out at 180s — increase to 240000ms
- Update WF_SCRIPT_GENERATE.json, re-import to n8n via PUT /api/v1/workflows/{ID}

Pattern B: script_pass_scores not updating in Supabase (JSONB write failure)
- Cause: The read-merge-write Code node used a stale reference to script_pass_scores
- Fix: In each merge Code node, ensure the read uses the ORIGINAL $('wf-sg-read-topic').first().json[0].script_pass_scores (not a later PATCH response which returns minimal/empty)
- The source of truth for existing scores is always the initial read-topic node output

Pattern C: Bulk scene INSERT returned 409 or 422 (duplicate scene_id)
- Cause: topic was retried without deleting old scenes first
- Fix: Ensure wf-sg-delete-scenes runs BEFORE wf-sg-insert-scenes, and the DELETE returns 204 before INSERT fires
- If connections are correct but still failing: add a Wait node (2000ms) between delete and insert

Pattern D: PassTracker not showing live pass scores (Realtime not updating)
- Cause: Supabase topics table may not have REPLICA IDENTITY FULL set
- Fix: SSH to VPS, run: docker exec supabase-db-1 psql -U postgres -d postgres -c "ALTER TABLE topics REPLICA IDENTITY FULL;"
- Verify: SELECT relreplident FROM pg_class WHERE relname='topics'; — should return 'f' (full)

Pattern E: Production not triggered after Gate 2 approval (topics stuck at script_approved)
- Cause: WF_SCRIPT_APPROVE's self-chain POST to /webhook/production/trigger returned error
- Fix: Check the N8N_WEBHOOK_BASE env var is set in n8n container. The URL should be https://n8n.srv1297445.hstgr.cloud/webhook — not /webhook/webhook
- Check: docker exec n8n-n8n-1 env | grep N8N_WEBHOOK_BASE

Pattern F: Scenes have no visual_type (all null)
- Cause: wf-sg-claude-visual Claude response parsing failed
- Fix: In wf-sg-build-scenes Code node, add fallback: if visual_type is null, assign 'static_image' as default. Then in the visual assignment step, improve the Claude prompt to return a flat JSON object { scene_001: "static_image", scene_002: "i2v", ... } rather than an array.

Pattern G: Script Review page shows blank (no script content renders)
- Cause: script_json structure doesn't match what ScriptContent.jsx expects
- Fix: Inspect topics.script_json in Supabase. Check dashboard/src/components/script/ScriptContent.jsx for expected structure. Common issue: expects script_json.scenes array; if stored as { chapters: [...] } instead, update the Code node in wf-sg-build-scenes to match.

Pattern H: Google Drive OAuth expired (TTS upload fails with 401)
- Fix: n8n Settings → Credentials → GoogleDriveAccount → Reconnect
- No workflow changes needed
