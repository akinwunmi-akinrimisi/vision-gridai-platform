  <name>Task 1: Write WF_SCRIPT_GENERATE workflow JSON</name>
  <files>workflows/WF_SCRIPT_GENERATE.json</files>
  <action>
Create workflows/WF_SCRIPT_GENERATE.json as a complete n8n workflow JSON. Use node IDs prefixed wf-sg-*. Use exact credential names as specified in interfaces.

The workflow has 34 nodes in 6 groups. responseMode must be "responseNode" so the webhook returns immediately while the pipeline runs asynchronously.

GROUP A — Webhook + Init (9 nodes):
1. wf-sg-webhook: Webhook Trigger, path "script/generate", responseMode "responseNode", httpMethod POST
2. wf-sg-validate: Code — parse body.topic_id, throw if missing, return { topic_id }
3. wf-sg-respond: Respond to Webhook — respondWith json, body { success: true, message: "Script generation started" }, HTTP 200. This node fires immediately after validate so the caller gets an instant response.
4. wf-sg-read-topic: HTTP GET /rest/v1/topics?id=eq.TOPIC_ID&select=*,avatars(*) using SUPABASE_URL env var
5. wf-sg-read-prompts: HTTP GET /rest/v1/prompt_configs?project_id=eq.PROJECT_ID&is_active=eq.true (project_id from topic)
6. wf-sg-read-project: HTTP GET /rest/v1/projects?id=eq.PROJECT_ID&select=*
7. wf-sg-init: Code — validate that script_pass1/pass2/pass3/evaluator/visual_director prompt types all exist in prompts array. Extract each into named vars. Set attempt = (topic.script_attempts||0)+1. Avatar = topic.avatars[0]. Return { topic, avatar, project, prompts, attempt, topic_id, project_id }
8. wf-sg-patch-scripting: HTTP PATCH topics — set status=scripting, script_attempts=attempt, updated_at=new Date().toISOString()
9. wf-sg-log-started: HTTP POST production_log — { project_id, topic_id, stage: script_generation, action: started, details: { attempt } }

Connections: webhook->validate, validate->respond (output 0) AND validate->read-topic (output 0 also fans out — n8n supports multiple connections from one output). read-topic->read-prompts->read-project->init->patch-scripting->log-started

GROUP B — Pass 1 Foundation (7 nodes):
10. wf-sg-build-pass1: Code — replace all 15 template variables in prompts.pass1 text using String.replace. Variables: niche_expertise_profile, seo_title, narrative_hook, avatar_name_age, pain_point, occupation_income, life_stage, spending_profile, knowledge_level, emotional_driver, online_hangouts, objection, dream_outcome, playlist_angle, key_segments. All sourced from init output. Return { prompt }
11. wf-sg-claude-pass1: HTTP POST Anthropic API — model claude-sonnet-4-6, max_tokens 8192, timeout 120000, onError continueRegularOutput. System field: project.niche_system_prompt. Messages: [{ role: user, content: prompt }]
12. wf-sg-check-pass1-err: Code — error check pattern. Extract pass1_text from content[0].text if no error.
13. wf-sg-if-pass1-err: IF node — $json.isError === true → true branch to error group. false branch continues.
14. wf-sg-eval-pass1-prompt: Code — build evaluator prompt from prompts.evaluator, inject pass1_text. Return { eval_prompt, pass1_text }
15. wf-sg-claude-eval1: HTTP POST Anthropic API — max_tokens 2048, timeout 60000, onError continueRegularOutput. User message: eval_prompt. Parse score from response content[0].text as JSON.
16. wf-sg-merge-pass1: Code — parse evaluator JSON response. Get score/dimensions/feedback. Read existing script_pass_scores from $('wf-sg-read-topic').first().json[0].script_pass_scores||{}. Set existing.pass1 = { score, dimensions, feedback, word_count: pass1_text.split(' ').length, timestamp }. Return { script_pass_scores: JSON.stringify(existing), pass1_text, pass1_score: score }
17. wf-sg-patch-p1: HTTP PATCH topics — script_pass_scores=$json.script_pass_scores (triggers Realtime PassTracker)
18. wf-sg-log-pass1: HTTP POST production_log — action pass1_complete, details { score, word_count }

GROUP C — Pass 2 Depth (7 nodes):
19. wf-sg-build-pass2: Code — inject same 15 variables into prompts.pass2. Return { prompt, pass1_text } where pass1_text comes from wf-sg-merge-pass1
20. wf-sg-claude-pass2: HTTP POST Anthropic — max_tokens 12288, timeout 180000, onError continueRegularOutput. Messages: [{ role: user, content: prompt }, { role: assistant, content: pass1_text }]. The assistant turn provides Pass 1 as context for depth pass.
21. wf-sg-check-pass2-err: Code — same error check, extract pass2_text
22. wf-sg-if-pass2-err: IF — same routing
23. wf-sg-merge-pass2: Code — same score parse, merge existing.pass2. Read fresh script_pass_scores from wf-sg-patch-p1 or wf-sg-read-topic. Return { script_pass_scores, pass2_text, pass2_score }
24. wf-sg-patch-p2: HTTP PATCH topics — script_pass_scores (Realtime update)
25. wf-sg-log-pass2: HTTP POST production_log — action pass2_complete

GROUP D — Pass 3 Resolution (6 nodes):
26. wf-sg-build-pass3: Code — inject variables into prompts.pass3. Include short summaries: pass1_text.substring(0,2000) and pass2_text.substring(0,2000) as context (NOT full text — avoids context overflow). Return { prompt, pass1_text, pass2_text }
27. wf-sg-claude-pass3: HTTP POST Anthropic — max_tokens 8192, timeout 120000, onError continueRegularOutput
28. wf-sg-check-pass3-err: Code — extract pass3_text
29. wf-sg-if-pass3-err: IF — error routing
30. wf-sg-merge-pass3: Code — merge existing.pass3. Return { script_pass_scores, pass1_text, pass2_text, pass3_text, pass3_score }
31. wf-sg-patch-p3: HTTP PATCH topics — script_pass_scores
32. wf-sg-log-pass3: HTTP POST production_log — action pass3_complete

GROUP E — Combined Eval + Visual Assignment + Store (8 nodes):
33. wf-sg-combined: Code node that does:
  a) Concatenate pass1+pass2+pass3 texts
  b) Build combined evaluator prompt
  c) Return { combined_text, eval_prompt, pass1_text, pass2_text, pass3_text, force_passed: false, attempt }
34. wf-sg-claude-combined: HTTP POST Anthropic — max_tokens 4096, timeout 90000. Evaluate combined script.
35. wf-sg-merge-combined: Code — parse combined score. Get existing script_pass_scores (from wf-sg-read-topic base). Merge existing.combined = { score, dimensions, feedback }. If combined_score < 7.0 AND attempt >= 3: force_passed = true. Return { script_pass_scores, combined_score, force_passed, combined_text, pass1_text, pass2_text, pass3_text }
36. wf-sg-patch-combined: HTTP PATCH topics — script_quality_score, script_evaluation (JSON string of { score, dimensions, feedback, timestamp }), script_pass_scores, script_force_passed (only if force_passed=true)
37. wf-sg-build-visual-prompt: Code — build visual_director prompt from prompts.visual_director. Inject full list of scenes parsed from combined_text (each with scene_id, narration_text, image_prompt). Return { visual_prompt, raw_scenes_array }
38. wf-sg-claude-visual: HTTP POST Anthropic — max_tokens 4096, timeout 90000. Returns JSON mapping scene_ids to visual_types.
39. wf-sg-build-scenes: Code — parse scenes from pass texts. Each scene: { topic_id, project_id, scene_number, scene_id (scene_001 padded), narration_text, image_prompt, visual_type (from visual assignment), emotional_beat, chapter }. Build script_json and script_metadata. Compute word_count, scene_count. Return { scenes_array, script_json, script_metadata, word_count, scene_count }
40. wf-sg-patch-script: HTTP PATCH topics — script_json, script_metadata, word_count, scene_count, updated_at
41. wf-sg-delete-scenes: HTTP DELETE /rest/v1/scenes?topic_id=eq.TOPIC_ID — clears prior scenes on retry
42. wf-sg-insert-scenes: HTTP POST /rest/v1/scenes — body: JSON.stringify(scenes_array). Prefer: return=minimal.
43. wf-sg-patch-review: HTTP PATCH topics — status=review, updated_at. This is the Realtime signal to dashboard.
44. wf-sg-log-completed: HTTP POST production_log — action completed, details { word_count, scene_count, quality_score, force_passed, attempts }

GROUP F — Error Handler (3 nodes, wired from all IF true branches):
45. wf-sg-collect-error: Code — extract error_message from $json. Pass topic_id and project_id from $('wf-sg-init') reference.
46. wf-sg-patch-failed: HTTP PATCH topics — status=failed, error_log=error_message, updated_at
47. wf-sg-log-failed: HTTP POST production_log — action failed, details { error, step: which group triggered this }
