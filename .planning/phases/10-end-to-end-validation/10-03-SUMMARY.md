## Bugs Found and Fixed

### Bug #13: [P1] WF_TOPICS_GENERATE max_tokens truncation
- **Cause**: `max_tokens: 16384` too small for 25 topics + 25 full avatars
- **Fix**: Increased to `32768` in Build Claude Body node + added `anthropic-beta: output-128k-2025-02-19` header to Claude HTTP request

### Bug #14: [P1] WF_TOPICS_ACTION Switch v3 routing broken — all actions go to approve branch
- **Cause**: Switch node v3 in `rules` mode had malformed conditions (missing `options.caseSensitive`, `options.typeValidation`, `combinator` fields; stale `outputIndex` from v2). All actions fell through to output 0 (approve).
- **Fix**: Changed Switch to `expression` mode: `={{ {'approve': 0, 'reject': 1, 'refine': 2, 'edit': 3, 'edit_avatar': 4}[$json.action] }}`
- **Impact**: Had to reset 3 incorrectly-approved topics back to pending

## Key Learnings
- n8n Switch v3 in `rules` mode requires proper `conditions.options` and `conditions.combinator` fields. Missing these causes silent fallthrough.
- Switch `expression` mode is simpler and more reliable for action routing.
- Claude Sonnet 4.6 needs `anthropic-beta: output-128k-2025-02-19` header for >16K output tokens.
- n8n caches active workflows in memory — must restart container after MCP partial updates.
