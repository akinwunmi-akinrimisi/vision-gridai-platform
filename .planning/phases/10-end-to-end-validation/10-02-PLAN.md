2. wf-sa-validate: Code — extract topic_id from body. Throw if missing. Return { topic_id }.
3. wf-sa-respond: Respond to Webhook — { success: true }, HTTP 200. Fires immediately.
4. wf-sa-read-topic: HTTP GET /rest/v1/topics?id=eq.TOPIC_ID&select=id,project_id,seo_title — get project_id for production_log
5. wf-sa-patch-approved: HTTP PATCH topics — { script_review_status: "approved", status: "script_approved", updated_at: new Date().toISOString() }
6. wf-sa-trigger-production: HTTP POST $env.N8N_WEBHOOK_BASE/production/trigger — body { topic_id }. Header: Authorization: Bearer $env.DASHBOARD_API_TOKEN. This chains Gate 2 → production pipeline (WF_WEBHOOK_PRODUCTION is already deployed).
7. wf-sa-log: HTTP POST production_log — { project_id: $('wf-sa-read-topic').first().json[0].project_id, topic_id, stage: "script_review", action: "approved", details: { title: topic.seo_title } }
8. wf-sa-error-patch: HTTP PATCH topics — status=script_approve_failed (wired from any node onError, or use continueRegularOutput + IF check if needed)
