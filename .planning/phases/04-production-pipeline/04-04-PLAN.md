  <action>
Create a complete n8n workflow JSON file for production webhook endpoints. Follow the exact same structure as WF_WEBHOOK_TOPICS_ACTION.json (read it for the auth pattern, node structure, and response format).

**Webhook endpoints (each is a separate Webhook Trigger node in the same workflow, routed via a Switch node on the path):**

1. **POST /webhook/production/trigger** — Entry point for starting production
   - Validate: topic exists, script_review_status='approved', status IN ('script_approved', 'stopped')
   - Create Google Drive subfolder structure for the topic: audio/, images/, video_clips/, captions/, final/ (use n8n Google Drive node or HTTP Request to Drive API)
   - Store subfolder IDs in topics.drive_subfolder_ids JSONB
   - Update topic.status = 'audio', topic.last_status_change = now()
   - Log to production_log: stage='production', action='started'
   - Fire TTS workflow via HTTP POST to /webhook/production/tts with { topic_id }
   - Return { success: true, data: { topic_id, status: 'audio' } }

2. **POST /webhook/production/trigger-batch** — Batch trigger
   - Receive { topic_ids: [...] }
   - Set first topic to 'producing', rest to 'queued' (ordered by topic_number)
   - Fire trigger for first topic only

3. **POST /webhook/production/stop** — Stop production
   - Set topic.status = 'stopped'
   - Log to production_log: stage='production', action='stopped'

4. **POST /webhook/production/resume** — Resume stopped production
   - Set topic.status back to its previous production stage (read from production_log last 'started' action)
   - Re-fire the appropriate stage webhook

5. **POST /webhook/production/restart** — Restart from scratch
   - Delete all scene assets (reset audio_status, image_status, video_status, clip_status to 'pending', clear URLs and drive IDs)
   - Reset topic.status = 'audio', clear cost_breakdown
   - Fire TTS workflow

6. **POST /webhook/production/retry-scene** — Retry single failed scene
   - Reset the failed status field(s) to 'pending'
   - Determine which stage failed and fire appropriate workflow for just that scene

7. **POST /webhook/production/retry-all-failed** — Batch retry all failed scenes
   - Reset all failed scenes to 'pending'
   - Group by failed stage and fire appropriate workflows

8. **POST /webhook/production/skip-scene** — Skip scene
   - Set scene.skipped = true, scene.skip_reason = reason
   - Check if all scenes now complete/skipped -> if so, fire assembly

9. **POST /webhook/production/skip-all-failed** — Skip all failed
   - Set all failed scenes to skipped=true

10. **POST /webhook/production/edit-retry-scene** — Edit image prompt and retry
    - Update scene.image_prompt with new prompt
    - Reset failed status to 'pending'
    - Fire appropriate visual generation for that scene

11. **POST /webhook/production/assemble** — Manual assembly trigger
    - Validate all scenes complete or skipped
    - Fire captions + assembly workflow

Each endpoint: auth check first, validate inputs, perform Supabase operations, return JSON envelope. Use n8n HTTP Request nodes for Supabase REST calls. Use credentials references (not hardcoded keys).

The JSON structure must be valid n8n workflow format with nodes, connections, and settings.

<task type="auto">
  <name>Task 2: Build WF_TTS_AUDIO.json - Sequential TTS with master timeline</name>
  <files>workflows/WF_TTS_AUDIO.json</files>
  <action>
Create a complete n8n workflow JSON for TTS audio generation. This is the first production stage and sets the master clock.

**Workflow flow:**

1. **Webhook Trigger** — Receives { topic_id } from production trigger webhook
2. **Auth Check** — IF node validates DASHBOARD_API_TOKEN header
3. **Load Scenes** — HTTP Request GET to Supabase: scenes?topic_id=eq.{topic_id}&audio_status=eq.pending&order=scene_number.asc
4. **Log Start** — INSERT to production_log: stage='audio', action='started'
5. **Loop Over Scenes** — n8n SplitInBatches node (batch size 1 for sequential):

   For each scene:
   a. **Check Stop** — GET topics?id=eq.{topic_id}&select=status. IF status='stopped', exit loop.
   b. **TTS Call** — HTTP Request POST to Google Cloud TTS:
      - URL: https://texttospeech.googleapis.com/v1/text:synthesize
      - Authentication: predefinedCredentialType (googleApi)
      - Body: { input: { text: scene.narration_text }, voice: { languageCode: "en-US", name: "en-US-Chirp3-HD-Charon" }, audioConfig: { audioEncoding: "MP3", sampleRateHertz: 24000 } }
   c. **Save to File** — Write base64 audio to /tmp/production/{topic_id}/audio/scene_{scene_number padded 3}.mp3 (use n8n Write Binary File node after decoding base64)
   d. **FFprobe Duration** — Execute Command: `ffprobe -v quiet -show_entries format=duration -of csv=p=0 /tmp/production/{topic_id}/audio/scene_{NNN}.mp3`
      - Parse output to get duration_seconds, multiply by 1000 for duration_ms
   e. **Compute Timeline** — Code node: start_time_ms = previous scene's end_time_ms (or 0 for first), end_time_ms = start_time_ms + duration_ms
   f. **Upload to Drive** — Google Drive Upload node: file from /tmp/, destination = audio subfolder from topics.drive_subfolder_ids
   g. **Update Scene** — PATCH scenes?id=eq.{scene_id}: audio_status='uploaded', audio_duration_ms, audio_file_drive_id, audio_file_url, start_time_ms, end_time_ms
   h. **Update Cost** — PATCH topics: increment cost_breakdown.tts (TTS cost per scene ~$0.002), update total_cost
   i. **Error Handling** — On TTS or upload failure: retry up to 3 times with exponential backoff (n8n Wait node: 30s, 60s, 120s). If all fail: set scene.audio_status='failed', scene.error_log, continue to next scene.

6. **On Complete** — After all scenes processed:
   a. Update topic.status = 'images'
   b. Log to production_log: stage='audio', action='completed', details: { scenes_completed, scenes_failed, total_duration_ms }
   c. Fire 3 parallel visual webhooks via HTTP POST:
      - /webhook/production/images with { topic_id }
      - /webhook/production/i2v with { topic_id }
      - /webhook/production/t2v with { topic_id }
   d. These visual webhook endpoints will be handled in Plan 04-05

The JSON must include proper n8n error handling: continueOnFail on critical nodes, error output connections, retry logic via Wait + IF nodes.

Use credential references for Google Cloud (googleApi type) and Supabase (custom header auth). Do NOT hardcode any API keys.
