  <action>
Create an n8n workflow JSON for image generation using the sliding window batch pattern.

**Workflow flow:**

1. **Webhook Trigger** — Receives { topic_id } from TTS completion or production webhook
2. **Auth Check** — IF node validates DASHBOARD_API_TOKEN
3. **Load Scenes** — GET scenes?topic_id=eq.{topic_id}&visual_type=eq.static_image&image_status=eq.pending&order=scene_number.asc
4. **Log Start** — INSERT production_log: stage='images', action='started', details: { scene_count }

5. **Sliding Window Loop** — Implement using n8n Code node (JavaScript):
   ```
   Active slots: [] (max 5)
   Pending queue: all loaded scenes

   While pending.length > 0 OR active.length > 0:
     // Check stop
     GET topics?id=eq.{topic_id}&select=status
     IF status='stopped' -> exit

     // Fill slots
     While active.length < 5 AND pending.length > 0:
       scene = pending.shift()
       POST createTask to Kie.ai with scene.image_prompt
       active.push({ scene, taskId })

     // Poll active
     Wait 3 seconds
     For each slot in active:
       GET recordInfo?taskId=slot.taskId
       IF state='success':
         Download image from resultUrl
         Upload to Google Drive images subfolder
         PATCH scene: image_status='uploaded', image_url, image_drive_id
         Update topics.cost_breakdown.images += 0.032
         Remove from active
       IF state='fail':
         IF retry_count < 3:
           Re-submit with backoff
         ELSE:
           PATCH scene: image_status='failed', error_log
           Remove from active
       // 'waiting','queuing','generating' -> continue polling
   ```

   Note: The n8n Code node executes JavaScript. Use `$getWorkflowStaticData('global')` for persistent state across executions, or implement the loop entirely within the Code node using async/await with fetch calls.

   Alternative implementation: Use a Loop node approach:
   - SplitInBatches with batches of 5 scenes
   - For each batch: POST all 5 createTask calls
   - Loop: poll all active tasks, handle completions, continue until all done
   - Process next batch

6. **Completion Check** — After all scenes processed:
   a. Log to production_log: stage='images', action='completed', details: { completed, failed }
   b. **Parallel completion sync:** Count all visual scenes (static_image + i2v + t2v) for this topic where status is 'uploaded' or skipped=true. If count equals total visual scenes, atomically claim assembly via PATCH topics?id=eq.{topic_id}&assembly_status=eq.pending body: { assembly_status: 'starting' }. If row returned, fire /webhook/production/assembly. Otherwise exit (another workflow will fire assembly).

Use Kie.ai credential reference for Authorization header. Use Supabase credential reference for API operations. No hardcoded keys.

Per scene cost: $0.032 (Seedream 4.5 basic quality). Update topics.cost_breakdown.images and topics.total_cost incrementally.
