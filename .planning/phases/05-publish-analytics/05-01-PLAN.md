Implement MetadataPanel.jsx (left sticky panel):

**Display mode (isEditing=false):**
- Title: topic.seo_title displayed as h2
- Description: auto-generated from template (hook paragraph from topic.narrative_hook, chapter timestamps from scenes, hashtags from tags, affiliate disclaimer placeholder). Show as formatted preview with line breaks.
- Tags: rendered as removable chip/pill badges (flex flex-wrap gap-1.5, each tag in rounded-full bg-primary/10 text-primary px-3 py-1 text-xs font-medium)
- Chapters: auto-generated list from scenes — group scenes by chapter field, for each chapter show timestamp (HH:MM:SS from first scene's start_time_ms) + chapter name. Use generateChapters() utility (defined inline or in a helper).
- Playlist assignment: text display showing playlist group number/name from topic.playlist_group
- "Edit Metadata" button (Pencil icon from lucide)

**Edit mode (isEditing=true):**
- Click "Edit Metadata" transforms all display fields to editable inputs IN-PLACE (not a modal, not a separate view):
  - Title: `<input type="text">` with current value
  - Description: `<textarea>` with raw text (not formatted preview)
  - Tags: chip badges with X remove button + text input at end (Enter to add new tag)
  - Chapters: editable list — each row has timestamp input + chapter name input
  - Playlist: dropdown select with playlist 1, 2, 3 options
  - Thumbnail prompt: textarea showing the image prompt used for thumbnail
- Save / Cancel buttons appear at bottom of panel
- Save calls useUpdateMetadata mutation with all edited fields, exits edit mode on success
- Cancel resets local state to original values, exits edit mode

**Action buttons (always visible at bottom):**
- "Approve" button (green, CheckCircle icon) — opens PublishDialog
- "Reject" button (red outline, XCircle icon) — opens RejectDialog
- "Retry Upload" button (amber, RefreshCw icon) — only visible when publish_progress='failed', calls useRetryUpload
- "Regenerate Thumbnail" button (outline, Image icon) — calls useRegenerateThumbnail with current thumbnail prompt text; show loading spinner while pending
- Buttons hidden when topic is published (read-only mode)

Implement PublishDialog.jsx:
- Uses ConfirmDialog as wrapper
- Title: "Publish Video"
- Content shows: topic title, total cost, scene count, quota status ("X uploads remaining today" from useQuotaStatus)
- Three radio-style options with descriptions:
  1. "Publish Now" — "Upload as public video immediately" (default selected)
  2. "Schedule" — "Upload as private, transition to public at scheduled time". When selected, show `<input type="datetime-local">` for schedule time picker
  3. "Approve Only" — "Mark approved without uploading. Publish later from batch publish."
- If remaining quota is 0: show amber warning "Daily quota reached. Video will upload as private and transition to public tomorrow."
- Confirm button text changes: "Publish Now" / "Schedule Upload" / "Approve" based on selection
- onConfirm calls useApproveVideo with { topic_id, action: selected_option, schedule_time }

Implement RejectDialog.jsx:
- Uses ConfirmDialog as wrapper (variant="danger")
- Title: "Reject Video"
- Feedback textarea: "What needs to change?" (required, minimum 10 chars)
- Rollback stage radio options:
  1. "Assembly only" — "Re-run FFmpeg with existing assets (free)"
  2. "Visuals + assembly" — "Keep audio, redo images/video/assembly (~$15)"
  3. "Full re-production" — "Start from TTS (~$17)"
- Each option shows estimated cost in parentheses
- Confirm button: "Reject & Rollback" (red)
- onConfirm calls useRejectVideo with { topic_id, feedback, rollback_stage }
