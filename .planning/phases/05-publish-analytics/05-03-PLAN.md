Create WF_WEBHOOK_PUBLISH.json — single n8n workflow with multiple webhook endpoints (same pattern as WF_WEBHOOK_PRODUCTION.json):

**Webhook endpoints (each a separate Webhook node in the workflow):**

1. `/webhook/video/approve` (POST):
   - Auth: check DASHBOARD_API_TOKEN header
   - Read body: topic_id, action ('publish_now' | 'schedule' | 'approve_only'), schedule_time
   - If action='approve_only': PATCH topics set video_review_status='approved', return success
   - If action='publish_now': PATCH topics set video_review_status='approved', status='publishing', publish_progress='uploading_video'. Then Execute Workflow node triggers WF_YOUTUBE_UPLOAD with topic_id and privacy='public'
   - If action='schedule': PATCH topics set video_review_status='approved', status='scheduled', scheduled_publish_at=schedule_time. Execute WF_YOUTUBE_UPLOAD with topic_id and privacy='private'

2. `/webhook/video/reject` (POST):
   - Auth check
   - Read body: topic_id, feedback, rollback_stage
   - PATCH topics set video_review_status='rejected', video_review_feedback=feedback
   - Based on rollback_stage:
     - 'assembly': set status='assembly', assembly_status='pending'
     - 'visuals': set status='images', images_progress='pending', i2v_progress='pending', t2v_progress='pending', assembly_status='pending'
     - 'full': set status='audio', audio_progress='pending', images_progress='pending', i2v_progress='pending', t2v_progress='pending', assembly_status='pending'
   - Self-chain: trigger the appropriate production workflow based on rollback_stage

3. `/webhook/video/publish` (POST):
   - Auth check. Read topic_id.
   - PATCH topics set status='publishing', publish_progress='uploading_video'
   - Execute WF_YOUTUBE_UPLOAD with topic_id and privacy='public'

4. `/webhook/video/batch-publish` (POST):
   - Auth check. Read topic_ids array.
   - For each topic_id: PATCH set status='publishing', then Execute WF_YOUTUBE_UPLOAD sequentially (not parallel — quota)
   - Return { success: true, queued: topic_ids.length }

5. `/webhook/video/update-metadata` (POST):
   - Auth check. Read topic_id + metadata fields (seo_title, description, tags, chapters, playlist_group, thumbnail_prompt)
   - PATCH topics with provided fields (only update fields that are present in request)
   - Also PATCH script_metadata JSONB to update tags/description within it

6. `/webhook/thumbnail/regenerate` (POST):
   - Auth check. Read topic_id, prompt.
   - Call Kie.ai Seedream 4.5 with the prompt (same pattern as WF_IMAGE workflow)
   - Upload result to Google Drive
   - PATCH topics set thumbnail_url to new URL
   - Return { success: true, thumbnail_url }

7. `/webhook/video/retry-upload` (POST):
   - Auth check. Read topic_id.
   - PATCH topics set publish_progress='uploading_video'
   - Execute WF_YOUTUBE_UPLOAD with topic_id

8. `/webhook/video/schedule` (POST):
   - Auth check. Read topic_id, schedule_time.
   - PATCH topics set scheduled_publish_at=schedule_time
   - Return success

Create WF_YOUTUBE_UPLOAD.json — the YouTube upload workflow:

**Trigger:** Execute Workflow (called from WF_WEBHOOK_PUBLISH)
**Input:** topic_id, privacy_status ('public' or 'private')

**Step 1: Load topic data**
- GET from topics table: seo_title, script_metadata (tags, description), drive_video_url, thumbnail_url, playlist_group
- GET from projects table: youtube_channel_id, youtube_playlist1_id, youtube_playlist2_id, youtube_playlist3_id
- GET from scenes table: chapters data (for description chapter list)

**Step 2: Check quota**
- Count today's uploads: query production_log where stage='youtube_upload' and action='completed' and created_at >= today start (midnight PT)
- If count >= 6 AND privacy_status='public': override privacy_status to 'private', set topics.scheduled_publish_at to tomorrow

**Step 3: Generate YouTube metadata** (Code node)
- Build description from template: narrative_hook paragraph + chapter timestamps (HH:MM:SS format from scenes) + hashtags + standard disclaimer
- Prepare snippet: title (seo_title), description, tags (from script_metadata.tags), categoryId "27" (Education)
- Prepare status: privacyStatus, selfDeclaredMadeForKids=false, publishAt (if scheduled)

**Step 4: Initiate resumable upload** (HTTP Request node)
- POST https://www.googleapis.com/upload/youtube/v3/videos?uploadType=resumable&part=snippet,status
- Headers: Authorization Bearer from YouTube OAuth2 credential, Content-Type application/json, X-Upload-Content-Type video/*
- Body: { snippet, status }
- Extract Location header as upload_uri
- PATCH topics set publish_progress='uploading_video'

**Step 5: Download video from Drive** (HTTP Request node)
- GET https://www.googleapis.com/drive/v3/files/${drive_file_id}?alt=media
- Using Google Drive OAuth2 credential
- Save to binary data

**Step 6: Upload video to YouTube** (HTTP Request node)
- PUT to upload_uri from Step 4
- Content-Type: video/*
- Body: binary file from Step 5
- Response: video resource JSON with video ID
- On failure: retry 2x with exponential backoff (Wait node + IF node loop)
- PATCH topics set publish_progress='setting_metadata', youtube_video_id=response.id

**Step 7: Upload captions** (HTTP Request node)
- Build SRT content from scenes (Code node): scene_number, timestamps HH:MM:SS,mmm format, narration_text
- POST https://www.googleapis.com/upload/youtube/v3/captions?uploadType=resumable&part=snippet
- Metadata: { snippet: { videoId, language: 'en', name: 'English' } }
- Then PUT with SRT content
- PATCH topics set publish_progress='uploading_captions'

**Step 8: Set thumbnail** (HTTP Request node)
- Download thumbnail from thumbnail_url (HTTP Request)
- POST https://www.googleapis.com/upload/youtube/v3/thumbnails/set?videoId=${video_id}
- Body: thumbnail image binary
- PATCH topics set publish_progress='setting_thumbnail'

**Step 9: Add to playlist** (HTTP Request node)
- Determine playlist_id from project based on topic.playlist_group (1->youtube_playlist1_id, 2->youtube_playlist2_id, 3->youtube_playlist3_id)
- POST https://www.googleapis.com/youtube/v3/playlistItems?part=snippet
- Body: { snippet: { playlistId, resourceId: { kind: 'youtube#video', videoId } } }
- PATCH topics set publish_progress='assigning_playlist'

**Step 10: Finalize**
- PATCH topics: publish_progress='complete', status='published', youtube_url='https://youtube.com/watch?v=${video_id}', published_at=now()
- INSERT production_log: stage='youtube_upload', action='completed', details={ video_id, privacy_status }

**Error handling:**
- Each step wrapped in try/catch (continueOnFail on HTTP nodes)
- On any failure: PATCH topics set publish_progress='failed', error_log=error message, status='upload_failed'
- INSERT production_log: stage='youtube_upload', action='failed', details={ step, error }

All HTTP Request nodes use YouTube OAuth2 credential from n8n credential manager. Use the Supabase REST API pattern (apikey + Bearer headers) for all database operations.
