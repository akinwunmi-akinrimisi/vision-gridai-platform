  <action>
Create a single n8n workflow JSON file `workflows/WF_WEBHOOK_SETTINGS.json` with 3 webhook endpoints, following the same pattern as existing webhook workflows (WF_WEBHOOK_PUBLISH.json).

The workflow has 3 webhook trigger nodes, each routing to their handler:

**Endpoint 1: `/webhook/project/update-settings`**
- Webhook node: POST, path `project/update-settings`
- Extracts `project_id` and remaining fields from body
- HTTP Request node: PATCH to `{{$env.SUPABASE_URL}}/rest/v1/projects?id=eq.{{$json.project_id}}`
  - Headers: apikey + Authorization (Service Role Key) + Content-Type: application/json + Prefer: return=minimal
  - Body: spread all fields except project_id
- Respond to Webhook: `{ "success": true }`
- Error handler: Respond with `{ "success": false, "error": "..." }`

**Endpoint 2: `/webhook/prompts/update`**
- Webhook node: POST, path `prompts/update`
- Extracts `prompt_id` and `prompt_text` from body
- HTTP Request node: PATCH to `{{$env.SUPABASE_URL}}/rest/v1/prompt_configs?id=eq.{{$json.prompt_id}}`
  - Body: `{ "prompt_text": "..." }`
- Respond to Webhook: `{ "success": true }`

**Endpoint 3: `/webhook/prompts/revert`**
- Webhook node: POST, path `prompts/revert`
- Extracts `prompt_id` and `version` from body
- Step 1: HTTP Request GET to fetch the historical version: `{{$env.SUPABASE_URL}}/rest/v1/prompt_configs?project_id=eq.{{project_id}}&prompt_type=eq.{{prompt_type}}&version=eq.{{version}}&select=prompt_text`
- Step 2: HTTP Request GET to fetch current active version to get its version number
- Step 3: HTTP Request POST to insert new row with incremented version + the old prompt_text + is_active=true
- Step 4: HTTP Request PATCH to set old active row's is_active=false
- Respond to Webhook: `{ "success": true }`

**Endpoint 4: `/webhook/prompts/regenerate`**
- Webhook node: POST, path `prompts/regenerate`
- This is a placeholder stub that returns `{ "success": true, "message": "Regeneration requires Claude API - implement in Phase 2 niche research workflow" }` because actual prompt regeneration requires Claude API calls that are part of the niche research workflow (Phase 2). The button exists in the UI for when that workflow is connected.

Use n8n workflow JSON format with proper node positioning (x/y coordinates spaced 250px apart horizontally). Include `continueOnFail: true` on HTTP Request nodes for error handling.

Note: This single workflow file documents all 4 endpoints. In n8n, each webhook path is a separate trigger within the same workflow (or they can be separate workflows imported from this file). Use the same structure as existing WF_WEBHOOK files in the workflows/ directory.
