**ChapterAccordion.jsx** — Glass-card accordion per chapter:
- Props: `{ chapter, isExpanded, onToggle, children }`
- Chapter object shape: `{ name, scenes: [], wordCount }`
- Full-width button with glass-card styling: chapter name left, word count + scene count + chevron right
- Chevron rotates 180deg when expanded (transition-transform)
- Content renders children when expanded
- "Solid" feel per user spec — not flimsy dropdowns. Use `glass-card` base class, border, slight shadow.

**SceneRow.jsx** — Individual scene display:
- Props: `{ scene, isEditing, onStartEdit, onTogglePrompt, showPrompt }`
- VS Code-style line numbers: left margin, monospace font (`font-mono`), 3-digit zero-padded (`String(scene.scene_number).padStart(3, '0')`), muted color (`text-slate-300 dark:text-slate-600`)
- Narration text: 14-15px, line-height 1.7, `text-slate-700 dark:text-slate-300`
- Visual type badge below text: `badge-blue` for static_image, `badge-purple` for i2v, `badge-amber` for t2v
- Image prompt: hidden by default. On hover, show a subtle "eye" icon. On click, expand to show the image prompt text below narration.
- Subtle divider line between scenes (`border-b border-border/20 dark:border-white/[0.03]`)
- Click on the scene row (not badge area) opens inline edit mode (sets isEditing via parent callback)
- Group hover effect for the eye icon and click-to-edit hint

**SceneEditForm.jsx** — Inline edit form:
- Props: `{ scene, onSave, onCancel, onRegenPrompt }`
- Expands in-place below the scene row (per user decision: click-to-edit inline expand)
- Narration textarea (auto-height, min 3 rows)
- Image prompt textarea (auto-height, min 2 rows)
- Visual type dropdown (static_image, i2v, t2v) using a simple select element
- "Regen Prompt" button — marks this scene for batch prompt regeneration (does NOT trigger immediately; adds to batch set tracked by parent)
- Save and Cancel buttons. Save calls onSave with `{ scene_id, narration_text, image_prompt, visual_type }`.
- Track edit state locally so Realtime updates don't reset the form (per RESEARCH.md pitfall 5)

**ScriptToolbar.jsx** — Toolbar above chapter list:
- Props: `{ onExpandAll, onCollapseAll, editedSceneCount, onRegenPrompts, onSearch, passInfo }`
- Expand All / Collapse All buttons
- "Regenerate Edited Prompts ({count})" button — enabled only when editedSceneCount > 0, calls onRegenPrompts
- Search input: text search within script (filters scenes by narration text, client-side). When active, only matching scenes shown across all chapters.
- Pass indicator: informational badge showing "3-pass | Attempt {n}" — not interactive

**ScriptContent.jsx** — Container component for right column:
- Props: `{ scenes, topic, onSceneEdit, onRegenPrompts }`
- Groups scenes by chapter using `groupByChapter(scenes)` helper (from RESEARCH.md Pattern 5)
- Manages expanded chapter state: first chapter expanded by default, rest collapsed
- Manages which scene is being edited (editingSceneId state)
- Manages which scenes have been edited (editedSceneIds Set for batch regen)
- Manages image prompt visibility per scene
- Renders ScriptToolbar + ChapterAccordion list, each containing SceneRow/SceneEditForm
- Search filtering: when search query is active, flatten all chapters and show matching scenes only

**ScriptReview.jsx** — Complete rewrite replacing the existing mock scaffold:

- Use `useParams()` to get `id` (project) and `topicId` from route `/project/:id/topics/:topicId/script`
- Use `useScript(topicId)` for topic/script data
- Use `useScenes(topicId)` for scene data
- Use mutation hooks from useScriptMutations.js

**Full-width header bar** (above the two columns):
- Topic number badge, SEO title (`topic.seo_title`), playlist angle badge, status badge
- Prev/Next topic arrows: fetch approved topics for this project via `useTopics(id)` filtered to review_status='approved', find current topic's position, navigate with `useNavigate()` to prev/next topicId. Disable arrows at boundaries.
- Back arrow to return to Topic Review page

**Two-column layout** (per user decision):
- `grid grid-cols-1 lg:grid-cols-12 gap-4`
- Left: `lg:col-span-4` — ScorePanel (sticky with `lg:sticky lg:top-4 lg:self-start lg:max-h-[calc(100vh-6rem)] lg:overflow-y-auto`)
- Right: `lg:col-span-8` — ScriptContent (scrollable)

**State management:**
- `showRejectDialog` state for ConfirmDialog
- `showRefinePanel` state for SidePanel
- Reject: ConfirmDialog with optional feedback textarea (same pattern as topic reject). On confirm: call rejectScript mutation.
- Refine: ScriptRefinePanel (SidePanel) with instructions textarea + refinement history. On submit: call refineScript mutation.
- Approve: call approveScript mutation directly, show toast on success

**Empty states:**
- If topic has no script (script_json is null, status is 'approved' for topics): Show "Generate Script" CTA. Brief explanation: "This will generate a 3-pass script (~8-10 minutes). Each pass is independently scored." Generate button calls generateScript mutation. After clicking, show PassTracker component.
- If topic status is 'scripting': Show PassTracker with live progress from `topic.script_pass_scores`
- If topic has script_json: Show full two-column layout with script content
- If loading: Show skeleton loading state

**ForcePassBanner** — Render above ScriptContent when `topic.script_force_passed === true`

**Mobile responsive** (per user decision):
- Score panel collapses to compact top bar on mobile showing score + action buttons
- Script content goes full-width below
- Use `hidden lg:block` / `lg:hidden` for responsive switching

**ScriptRefinePanel.jsx** — SidePanel for whole-script refine:
- Uses existing SidePanel component as wrapper
- Props: `{ isOpen, onClose, topic, onSubmit }`
- Instructions textarea for refinement directions
- Refinement history display (from `topic.refinement_history` if it exists, showing past instructions + timestamps)
- Submit button triggers the refine mutation
- Pattern follows exactly from the existing RefinePanel.jsx for topics

**TopicCard.jsx** — Add script status badge and View Script link:
- After the existing status badge, add a script status badge when topic has a script:
  - `script_review_status === 'pending'` and `status === 'scripting'` -> amber badge "Generating"
  - `script_review_status === 'pending'` and `script_json` exists -> amber badge "Review"
  - `script_review_status === 'approved'` -> green badge "Script Approved"
  - `script_review_status === 'rejected'` -> red badge "Script Rejected"

- Add "View Script" button (FileText icon) in the action buttons area, only visible when topic is approved (review_status === 'approved'). Uses `Link` from react-router to navigate to `/project/${projectId}/topics/${topic.id}/script`.
- The "View Script" button replaces the Edit button position when script exists (Edit still shows for topics without scripts).
