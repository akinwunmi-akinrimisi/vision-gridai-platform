Create 4 simpler webhook stubs:

**script-approve.json** — Synchronous (returns immediately with result):
- Request: `{ topic_id }`
- Action: PATCH topics SET script_review_status='approved', status='approved' WHERE id=eq.{topic_id}
- Insert production_log entry: { stage: 'script_review', action: 'approved' }
- Response: `{ success: true, data: { topic_id, script_review_status: 'approved' } }`
- Note: "After approval, status is 'approved' which signals Phase 4 production pipeline can begin. Phase 3 does NOT wire production — it only sets status."

**script-reject.json** — Synchronous:
- Request: `{ topic_id, feedback }`
- Action: PATCH topics SET script_review_status='rejected', script_review_feedback=feedback WHERE id=eq.{topic_id}
- Insert production_log entry: { stage: 'script_review', action: 'rejected', details: { feedback } }
- Response: `{ success: true }`

**script-refine.json** — Async (returns immediately, works in background):
- Request: `{ topic_id, instructions }`
- Immediate response: `{ success: true, message: "Refinement started" }`
- Async steps:
  1. Read topic (with script_json, script_pass_scores)
  2. Identify weakest pass from script_pass_scores (lowest combined dimension)
  3. PATCH topic: status='scripting', script_review_status='refining'
  4. Regenerate ONLY the weakest pass with instructions injected as additional context
  5. Re-score the regenerated pass
  6. Re-run combined evaluation
  7. Update script_json with new scenes from regenerated pass
  8. Delete + re-insert affected scene rows in scenes table
  9. Append to refinement_history JSONB array: { instructions, timestamp, pass_regenerated, new_score }
  10. PATCH topic: status='review', script_review_status='pending'

**script-regen-prompts.json** — Async:
- Request: `{ topic_id, scene_ids: ["uuid1", "uuid2", ...] }`
- Immediate response: `{ success: true, message: "Prompt regeneration started" }`
- Async steps:
  1. Read scenes by IDs
  2. For each scene: call Claude with visual_director prompt + scene narration_text to generate new image_prompt
  3. PATCH each scene: image_prompt = new_prompt
  4. Insert production_log entry

All stubs should include:
- `webhook` object with path, method, request schema, response schema
- `description` explaining purpose
- `supabase_writes_summary` listing tables touched
- `error_handling` section
