  <action>
Create an n8n workflow JSON for the supervisor agent that runs on a 30-minute schedule.

**Workflow flow:**

1. **Schedule Trigger** — n8n Cron node: every 30 minutes (*/30 * * * *)

2. **Find Stuck Topics** — HTTP Request GET to Supabase:
   ```
   GET topics?status=in.(audio,images,assembling)&select=id,status,last_status_change,project_id,seo_title,topic_number
   ```
   Then filter in Code node: topics where (now - last_status_change) > 2 hours (7200000 ms)

3. **For Each Stuck Topic** — SplitInBatches:

   a. **Check for Failed Scenes** — GET scenes?topic_id=eq.{id}&or=(audio_status.eq.failed,image_status.eq.failed,video_status.eq.failed)

   b. **If Failed Scenes Exist** — Auto-retry:
      - HTTP POST to /webhook/production/retry-all-failed with { topic_id }
      - Log to production_log: stage='supervisor', action='auto_retry', details: { topic_id, failed_count, reason: 'stuck >2h' }
      - Wait 5 minutes (n8n Wait node)
      - Re-check: GET scenes for same topic with failed statuses
      - If still has failures:
        - PATCH topics: supervisor_alerted=true
        - Log to production_log: stage='supervisor', action='alerted', details: { topic_id, remaining_failures, message: 'Auto-retry did not resolve stuck pipeline' }

   c. **If No Failed Scenes** (stuck but no failures = likely stalled workflow):
      - PATCH topics: supervisor_alerted=true
      - Log to production_log: stage='supervisor', action='alerted', details: { topic_id, status, message: 'Topic stuck with no failed scenes - possible workflow stall' }
      - Attempt to re-fire the current stage webhook:
        - If status='audio': POST /webhook/production/tts with { topic_id }
        - If status='images': POST /webhook/production/images, /production/i2v, /production/t2v
        - If status='assembling': POST /webhook/production/assembly

4. **Reset Resolved Alerts** — Find topics where supervisor_alerted=true but status IN ('assembled', 'published', 'stopped'). Set supervisor_alerted=false for these (alert auto-dismisses when resolved).

5. **Summary Log** — If any actions taken, INSERT production_log: stage='supervisor', action='run_complete', details: { stuck_count, retried_count, alerted_count, resolved_count }

No auth check needed (cron trigger, not webhook). Use Supabase service role key for all operations.
