n8n webhook workflows follow this structure:
- Webhook trigger node (type: n8n-nodes-base.webhook) with httpMethod, path, responseMode
- IF node for Bearer token validation (compare header against $env.DASHBOARD_API_TOKEN)
- Respond to Webhook node for success response (JSON envelope)
- Respond to Webhook node for auth failure response (JSON envelope with error)

Response envelope format (from CONTEXT.md locked decision):
```json
{ "success": true, "data": {}, "error": null }
```

Auth pattern (from CONTEXT.md locked decision):
- Bearer token in Authorization header
- Token stored as DASHBOARD_API_TOKEN environment variable in n8n
- "Respond Immediately" mode on all webhook triggers

  <action>
    Create 4 n8n workflow JSON files in the `workflows/` directory. Each workflow follows the same pattern:

    **Common structure for all 4 workflows:**
    1. **Webhook Trigger node** (n8n-nodes-base.webhook):
       - `httpMethod`: "POST"
       - `path`: the endpoint path (see below per workflow)
       - `responseMode`: "responseNode" (this is how n8n implements "Respond Immediately" -- the workflow continues executing but the HTTP response is sent by a Respond to Webhook node immediately)
       - `options.allowedOrigins`: "*"

    2. **IF node** (n8n-nodes-base.if) for Bearer token validation:
       - Condition: Check if `{{ $json.headers.authorization }}` equals `Bearer {{ $env.DASHBOARD_API_TOKEN }}`
       - Use string comparison, case-sensitive

    3. **Respond to Webhook - Success** (n8n-nodes-base.respondToWebhook):
       - Connected to IF node "true" output
       - `respondWith`: "json"
       - `responseBody`: JSON envelope with `success: true`
       - Response includes relevant stub data per endpoint

    4. **Respond to Webhook - Unauthorized** (n8n-nodes-base.respondToWebhook):
       - Connected to IF node "false" output
       - `respondWith`: "json"
       - `responseBody`: `{ "success": false, "data": null, "error": "Unauthorized" }`
       - `options.responseCode`: 401

    **Per-workflow specifics:**

    **WF_WEBHOOK_STATUS.json** (path: "status"):
    - The health check endpoint. Returns platform status.
    - Success response data: `{ "status": "healthy", "version": "1.0.0", "services": { "n8n": "running", "supabase": "configured" } }`
    - This is the primary endpoint for testing the webhook layer works.

    **WF_WEBHOOK_PROJECT_CREATE.json** (path: "project/create"):
    - Stub for project creation. In Phase 2, this will trigger niche research.
    - Success response data: `{ "message": "Project creation queued", "stub": true }`
    - NOTE: This is a stub. The actual project creation logic (Phase A from Dashboard_Implementation_Plan.md) will be built in Phase 2.

    **WF_WEBHOOK_TOPICS_GENERATE.json** (path: "topics/generate"):
    - Stub for topic generation. In Phase 2, this will trigger the 25-topic generation workflow.
    - Success response data: `{ "message": "Topic generation queued", "stub": true }`

    **WF_WEBHOOK_TOPICS_ACTION.json** (path: "topics/action"):
    - Stub for topic approve/reject/refine. In Phase 2, this will handle Gate 1 actions.
    - Expects body with `{ "action": "approve|reject|refine", "topic_id": "..." }`
    - Success response data: `{ "message": "Topic action received", "action": "{{ $json.body.action }}", "stub": true }`

    **n8n workflow JSON format requirements:**
    - Each file must be a valid n8n workflow export (importable via n8n UI -> Settings -> Import from File)
    - Include `name`, `nodes` array, `connections` object, and `settings` object
    - Each node needs: `id`, `name`, `type`, `typeVersion`, `position` (x, y coordinates), `parameters`
    - Connections map output node names to input node names with source/target indices
    - Use n8n's standard node type versions: webhook v2, if v2, respondToWebhook v1
    - Set `settings.executionOrder` to "v1" for predictable execution

    **Important:** The webhook path in n8n becomes the URL suffix. So path "status" means the endpoint is `https://n8n.srv1297445.hstgr.cloud/webhook/status`. The Vite dev proxy in Plan 01 forwards `/webhook/*` to n8n, so the dashboard can call `/webhook/status` directly.
