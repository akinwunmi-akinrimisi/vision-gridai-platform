  <action>
    **useAuth.js hook:**
    - Session key `gridai_session` in localStorage, 30-day expiry
    - `login(pin)` hashes PIN with Web Crypto API SHA-256, compares against `import.meta.env.VITE_PIN_HASH`
    - `logout()` clears localStorage and sets isAuthenticated=false
    - On mount, checks localStorage for valid (non-expired) session
    - Returns `{ isAuthenticated, login, logout }`

    **NOTE on PIN hash storage:** CONTEXT.md locks "PIN stored as hash in Supabase settings table (changeable later)." For Phase 1, the PIN hash is stored as `VITE_PIN_HASH` env var as a simplification -- no `settings` table exists in the current schema. This is an acknowledged Phase 1 deviation. Migration to a Supabase `settings` table lookup is planned for Phase 2+ when the projects/settings infrastructure is built. The useAuth hook is designed so that swapping the hash source (env var -> Supabase fetch) requires changing only the `login()` function internals.

    **useTheme.js hook:**
    - Reads initial theme from localStorage key `gridai_theme`, defaults to `'light'`
    - Toggles `dark` class on `document.documentElement`
    - Persists preference to localStorage
    - Returns `{ isDark, toggle }`

    **PinGate.jsx:**
    - Full-screen split layout: left side dark background with branding (Vision GridAI logo, tagline "AI Video Production Platform"), right side white/dark PIN form
    - 4-6 digit PIN input (type="password", maxLength=6, centered, large font, monospace)
    - Submit button styled with CTA color (#F97316)
    - Wrong PIN: CSS shake animation on input, red "Wrong PIN" text below, auto-clear input
    - Clean, professional design per MASTER.md spacing/shadows
    - Unlimited retries, no lockout

    **ThemeToggle.jsx:**
    - Simple button with Sun/Moon Lucide icons
    - Uses useTheme() hook
    - Sun icon in dark mode, Moon icon in light mode

    **useAuth.test.js:**
    - Test login with correct PIN hash returns true and sets isAuthenticated
    - Test login with wrong PIN returns false
    - Test logout clears session
    - Test session persistence from localStorage
    - Mock `crypto.subtle.digest` since jsdom doesn't support Web Crypto -- OR use a simpler hash comparison for testability (compare directly against env var hash string)

    **Design system application across ALL components:**
    - Use Tailwind classes that map to design system tokens (not arbitrary values)
    - Light mode: bg-surface text-text, dark mode: dark:bg-surface-dark dark:text-slate-100
    - Cards: bg-card dark:bg-card-dark, rounded-xl, shadow-md
    - Borders: border-border dark:border-border-dark
    - All transitions: transition-all duration-200
