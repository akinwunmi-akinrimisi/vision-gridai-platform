  <action>
    **useRealtimeSubscription.js:**
    - Custom hook: `useRealtimeSubscription(table, filter, queryKeys)`
    - `table`: string (e.g., 'projects', 'topics', 'scenes')
    - `filter`: string or null (e.g., `project_id=eq.${id}`)
    - `queryKeys`: array of TanStack Query keys to invalidate (e.g., `[['projects'], ['topics', projectId]]`)
    - Creates a unique channel name: `${table}-${filter || 'all'}-${Date.now()}` (include timestamp to avoid collisions per Pitfall 3 from research)
    - Subscribes to `postgres_changes` for all events (`event: '*'`) on the given table with optional filter
    - On receiving a payload, calls `queryClient.invalidateQueries({ queryKey: key })` for each key in queryKeys
    - Stores channel ref with `useRef`
    - Cleanup in useEffect return: `supabase.removeChannel(channelRef.current)` (prevents memory leaks)
    - Dependencies: `[table, filter]` -- queryKeys intentionally excluded (caller should pass stable reference)
    - Also returns the channel subscription status for ConnectionStatus

    **ConnectionStatus.jsx:**
    - Small indicator component showing Supabase Realtime connection state
    - Subscribe to a heartbeat channel to track connection health
    - States: connected (green dot + "Connected"), reconnecting (yellow dot + "Reconnecting"), disconnected (red dot + "Disconnected")
    - Use `supabase.channel('heartbeat').subscribe((status) => { ... })` to track SUBSCRIBED/CLOSED/CHANNEL_ERROR states
    - Compact display: colored dot + text label
    - Cleanup on unmount

    **Sidebar.jsx update:**

    - This is a targeted modification -- read the existing Sidebar.jsx from Plan 01, only add ConnectionStatus import and render
