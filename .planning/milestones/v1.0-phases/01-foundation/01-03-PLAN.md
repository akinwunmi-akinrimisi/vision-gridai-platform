    **infra/n8n-env.sh** -- Shell script that documents and applies n8n timeout environment variables:
    ```bash
    #!/bin/bash
    # n8n Timeout Configuration for Vision GridAI
    # Apply: Add these to your n8n Docker container's environment variables
    # Either in docker-compose.yml environment section or via docker run -e flags

    # Execution timeout: 600 seconds (10 min) -- script generation can take 2-5 min
    # Default is 300s which is too short for 3-pass script generation
    export EXECUTIONS_TIMEOUT=600
    export EXECUTIONS_TIMEOUT_MAX=900

    # Webhook timeout: disable the default response timeout
    # We use "Respond Immediately" mode, but this prevents edge cases
    export N8N_DEFAULT_BINARY_DATA_MODE=filesystem

    # Process timeout for individual nodes (in seconds)
    export N8N_EXECUTIONS_PROCESS_TIMEOUT=600

    # Keep execution data for debugging (last 500 executions)
    export EXECUTIONS_DATA_PRUNE=true
    export EXECUTIONS_DATA_MAX_AGE=168

    echo "n8n environment variables configured."
    echo "Add these to your docker-compose.yml environment section or .env file."
    ```

    **infra/postgres-tune.sh** -- Shell script to tune PostgreSQL for the platform's workload:
    ```bash
    #!/bin/bash
    # PostgreSQL Tuning for Vision GridAI (Supabase self-hosted)
    # Apply: Run via psql or add to postgresql.conf in Supabase Docker volume
    #
    # Default max_connections = 100, which is not enough for:
    #   - Supabase Realtime (holds persistent connections per subscription)
    #   - n8n workflows (each HTTP Request node uses a connection)
    #   - Dashboard direct queries via Supabase JS client
    #   - Supabase PostgREST (connection pooling)

    PSQL_CMD="psql -h localhost -p 54321 -U postgres"

    echo "Current max_connections:"
    $PSQL_CMD -c "SHOW max_connections;"

    echo ""
    echo "To increase max_connections, edit postgresql.conf in the Supabase PostgreSQL Docker volume:"
    echo "  1. Find the volume: docker inspect supabase-db | grep -A5 Mounts"
    echo "  2. Edit postgresql.conf: max_connections = 200"
    echo "  3. Adjust shared_buffers = 256MB (25% of available RAM, adjust for your VPS)"
    echo "  4. Set work_mem = 8MB"
    echo "  5. Set effective_cache_size = 1GB (50-75% of available RAM)"
    echo "  6. Restart PostgreSQL container: docker restart supabase-db"
    echo ""
    echo "Alternatively, run these ALTER SYSTEM commands (applied on restart):"

    $PSQL_CMD -c "ALTER SYSTEM SET max_connections = 200;"
    $PSQL_CMD -c "ALTER SYSTEM SET shared_buffers = '256MB';"
    $PSQL_CMD -c "ALTER SYSTEM SET work_mem = '8MB';"
    $PSQL_CMD -c "ALTER SYSTEM SET effective_cache_size = '1GB';"

    echo ""
    echo "Restart PostgreSQL for changes to take effect:"
    echo "  docker restart supabase-db"
    ```

    **infra/docker-compose.override.yml** -- Docker Compose override with memory limits:
    ```yaml
    # Docker Compose Override for Vision GridAI
    # Usage: docker compose -f docker-compose.yml -f docker-compose.override.yml up -d
    #
    # Adjust memory limits based on your VPS RAM.
    # These values assume a 4GB+ VPS. Scale proportionally.

    version: "3.8"

    services:
      # n8n container -- needs memory for Node.js + workflow execution
      n8n:
        deploy:
          resources:
            limits:
              memory: 1536M
            reservations:
              memory: 512M
        environment:
          - EXECUTIONS_TIMEOUT=600
          - EXECUTIONS_TIMEOUT_MAX=900
          - N8N_EXECUTIONS_PROCESS_TIMEOUT=600
          - N8N_DEFAULT_BINARY_DATA_MODE=filesystem
          - EXECUTIONS_DATA_PRUNE=true
          - EXECUTIONS_DATA_MAX_AGE=168
          - NODE_OPTIONS=--max-old-space-size=1024

      # Supabase PostgreSQL -- needs memory for connections + query cache
      supabase-db:
        deploy:
          resources:
            limits:
              memory: 1024M
            reservations:
              memory: 256M
        command: >
          postgres
          -c max_connections=200
          -c shared_buffers=256MB
          -c work_mem=8MB
          -c effective_cache_size=512MB

      # Supabase Realtime -- WebSocket connections from dashboard
      supabase-realtime:
        deploy:
          resources:
            limits:
              memory: 512M
            reservations:
              memory: 128M

      # Supabase PostgREST -- REST API layer
      supabase-rest:
        deploy:
          resources:
            limits:
              memory: 256M
            reservations:
              memory: 64M
    ```
