Create `dashboard/src/lib/publishApi.js` following the same pattern as productionApi.js (import webhookCall from api.js, export named functions):
- `approveVideo(topicId, action, scheduleTime)` -> webhookCall('video/approve', { topic_id, action, schedule_time })
  - action is one of: 'publish_now', 'schedule', 'approve_only'
- `rejectVideo(topicId, feedback, rollbackStage)` -> webhookCall('video/reject', { topic_id, feedback, rollback_stage })
  - rollbackStage is one of: 'assembly', 'visuals', 'full'
- `publishVideo(topicId)` -> webhookCall('video/publish', { topic_id })
- `scheduleVideo(topicId, scheduleTime)` -> webhookCall('video/schedule', { topic_id, schedule_time })
- `batchPublish(topicIds)` -> webhookCall('video/batch-publish', { topic_ids })
- `regenerateThumbnail(topicId, prompt)` -> webhookCall('thumbnail/regenerate', { topic_id, prompt })
- `updateMetadata(topicId, fields)` -> webhookCall('video/update-metadata', { topic_id, ...fields })
- `retryUpload(topicId)` -> webhookCall('video/retry-upload', { topic_id })

Create `dashboard/src/lib/analyticsApi.js`:
- `refreshAnalytics(projectId)` -> webhookCall('analytics/refresh', { project_id })

Create `dashboard/src/hooks/useVideoReview.js`:
- `useVideoReview(topicId)` — useQuery fetching single topic + its scenes from Supabase. Query key: ['video-review', topicId]. Fetches topic from topics table by id, scenes from scenes table by topic_id ordered by scene_number. Returns { topic, scenes, isLoading, error }. Also calls useRealtimeSubscription on topics table filtered by id for live publish_progress updates.

Create `dashboard/src/hooks/usePublishMutations.js`:
- `useApproveVideo(projectId)` — useMutation calling approveVideo, optimistically sets video_review_status to the action type, invalidates ['topics', projectId] and ['video-review', topicId] on settle
- `useRejectVideo(projectId)` — useMutation calling rejectVideo, optimistically sets video_review_status to 'rejected', invalidates same
- `usePublishVideo(projectId)` — useMutation calling publishVideo, invalidates same
- `useBatchPublish(projectId)` — useMutation calling batchPublish, invalidates ['topics', projectId]
- `useRegenerateThumbnail(projectId)` — useMutation calling regenerateThumbnail, invalidates video-review
- `useUpdateMetadata(projectId)` — useMutation calling updateMetadata, invalidates video-review
- `useRetryUpload(projectId)` — useMutation calling retryUpload, invalidates same

Create `dashboard/src/hooks/useAnalytics.js`:
- `useAnalytics(projectId, timeRange)` — useQuery fetching all published topics from Supabase with yt_* fields. timeRange is one of '7d', '30d', '90d', 'all'. Filter by published_at within range. Query key: ['analytics', projectId, timeRange]. Returns { data: Topic[], isLoading, error }. Computes derived metrics in useMemo: totalViews, totalWatchHours, avgCtr, totalRevenue, totalLikes, totalComments, totalSubscribers, avgDuration, topPerformer (topic with most views).

Create `dashboard/src/hooks/useQuotaStatus.js`:
- `useQuotaStatus(projectId)` — useQuery fetching count of topics where status='published' and published_at is today (UTC). Returns { uploadsToday, remaining: 6 - uploadsToday, isLoading }. Query key: ['quota', projectId]. Refreshes every 60 seconds (refetchInterval: 60000).
