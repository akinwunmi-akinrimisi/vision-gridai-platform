<task type="auto">
  <name>Task 1: n8n analytics cron workflow</name>
  <files>workflows/WF_ANALYTICS_CRON.json</files>
  <action>
Create WF_ANALYTICS_CRON.json — n8n workflow with two triggers:

**Trigger 1: Schedule Trigger** — runs daily at 6:00 AM UTC (cron: 0 6 * * *)
**Trigger 2: Webhook** — `/webhook/analytics/refresh` (POST, auth with DASHBOARD_API_TOKEN)
- Read body: project_id (optional — if provided, refresh only that project; if not, refresh all)

**Step 1: Get published videos** (HTTP Request to Supabase)
- If project_id provided: GET topics where project_id=eq.{project_id} AND youtube_video_id is not null, select id, youtube_video_id, status, scheduled_publish_at
- If no project_id: GET all topics where youtube_video_id is not null

**Step 2: Handle private-to-public transitions** (Code node)
- Filter topics where status='scheduled' AND scheduled_publish_at <= now()
- Also filter topics where status='publishing' AND privacy was set to 'private' due to quota (check production_log)
- For each: call YouTube Data API videos.update to change privacyStatus to 'public'
  - PUT https://www.googleapis.com/youtube/v3/videos?part=status
  - Body: { id: youtube_video_id, status: { privacyStatus: 'public' } }
  - On success: PATCH topics set status='published', published_at=now()

**Step 3: Build analytics query** (Code node)
- Collect all youtube_video_ids into comma-separated string
- Build YouTube Analytics API request:
  - GET https://youtubeanalytics.googleapis.com/v2/reports
  - Query params:
    - ids=channel==MINE
    - startDate: 90 days ago (YYYY-MM-DD format)
    - endDate: today (YYYY-MM-DD)
    - metrics=views,estimatedMinutesWatched,averageViewDuration,averageViewPercentage,likes,comments,subscribersGained,estimatedRevenue,cpm,impressions
    - dimensions=video
    - filters=video=={comma-separated video IDs}
    - sort=-views
  - Use YouTube OAuth2 credential (must include yt-analytics.readonly scope)

**Step 4: Parse analytics response** (Code node)
- YouTube Analytics returns: { rows: [ [videoId, views, watchMinutes, avgDuration, avgPct, likes, comments, subsGained, revenue, cpm, impressions], ... ] }
- Map each row to an object with field names
- Calculate CTR from impressions and views (clicks/impressions — note: YouTube Analytics doesn't directly return CTR as a separate metric, compute from impressions click data or use annotationClickThroughRate if available; fallback: views/impressions * 100)

**Step 5: Write to Supabase** (SplitInBatches + HTTP Request)
- For each video in parsed results:
  - Find matching topic by youtube_video_id
  - PATCH topics table:
    - yt_views = views
    - yt_watch_hours = estimatedMinutesWatched / 60 (convert to hours)
    - yt_avg_view_duration = format avgDuration as "M:SS" string
    - yt_avg_view_pct = averageViewPercentage
    - yt_ctr = computed CTR
    - yt_impressions = impressions
    - yt_likes = likes
    - yt_comments = comments
    - yt_subscribers_gained = subscribersGained
    - yt_estimated_revenue = estimatedRevenue
    - yt_actual_cpm = cpm
    - yt_last_updated = now()

**Step 6: Log completion** (HTTP Request to Supabase)
- INSERT production_log: stage='analytics', action='completed', details={ videos_updated: count, timestamp: now() }

**Error handling:**
- If YouTube Analytics API returns 403 (scope issue): log warning in production_log with details about required yt-analytics.readonly scope
- If no published videos found: skip analytics pull, just handle transitions, return success
- Wrap entire flow in error handler: on failure, INSERT production_log with stage='analytics', action='failed'

**Response (for webhook trigger):**
- Return { success: true, videos_updated: count, transitions: transition_count }
