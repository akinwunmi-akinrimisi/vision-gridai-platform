# SOP 08 — Platform Render + QA Check + Gate 3 + Publish

## Purpose
Render platform-specific exports, run automated QA (13 checks), present for human review at Gate 3, then publish to YouTube with full metadata.

## Platform Export Profiles
| Platform | Resolution | CRF | Preset | Bitrate | Audio |
|----------|-----------|-----|--------|---------|-------|
| YouTube Long | 1920x1080 | 17-19 | slow | 12 Mbps | 192k AAC |
| YouTube Shorts | 1080x1920 | 18-20 | slow | 6 Mbps | 192k AAC |
| TikTok | 1080x1920 | 20-23 | medium | 4 Mbps | 128k AAC |
| Instagram | 1080x1920 | 20-23 | medium | 3.5 Mbps | 128k AAC |

All with `-movflags +faststart`. Each render stored in `renders` table.

**Do NOT use one export for all platforms.** Each has different CRF/bitrate targets.

## Automated QA — 13 Checks (WF_QA_CHECK)

### Visual (5)
1. Resolution matches target (FFprobe)
2. File size within expected range
3. No black frames (>2s) detected
4. Frame rate = 30fps
5. Aspect ratio correct (16:9 long / 9:16 short)

### Caption (3)
6. ASS/SRT file exists and is non-empty
7. No text overflow beyond safe area
8. Highlight words match narration text

### Audio (3)
9. Duration match: video vs sum of scene audio (tolerance ±500ms)
10. Loudness within target (-16 LUFS ±1)
11. No silence gaps >2s

### Platform (2)
12. H.264 codec + AAC audio + .mp4 container
13. `faststart` moov atom present

Results stored as JSONB in `renders.qa_results`. Dashboard shows pass/fail checklist.

## GATE 3 — Video Review (Pipeline Pauses)
Dashboard shows:
- Embedded video player (Google Drive URL)
- Thumbnail preview + "Regenerate" button
- YouTube metadata (title, description, tags) — editable
- PPS card (CF13 predicted performance score)
- 5-variant title picker (CF05 CTR title options)
- Thumbnail CTR score (CF06)
- QA checklist (13 items, pass/fail badges)
- Total production cost
- Platform selector checkboxes (YouTube, TikTok, Instagram)
- Upload visibility toggle (public/unlisted/private — default unlisted)

| Action | Effect |
|--------|--------|
| Approve & Publish | Upload immediately to selected platforms |
| Approve & Schedule | Set date/time, upload as private, auto-publish at scheduled time |
| Reject | Go back to specific stage for rework |
| Edit Metadata | Inline edit title/description/tags before publish |

## YouTube Upload
```
YouTube Data API v3 — resumable upload
  - Video file from Drive
  - Metadata from platform_metadata table
  - SRT captions via captions.insert
  - Thumbnail via thumbnails.set
  - Playlist assignment per topic's playlist_group
```

## Critical Rules
1. **Auto-pilot: NEVER publishes as public.** Always unlisted. Human changes visibility manually.
2. **YouTube API quota: 10,000 units/day.** Upload = 1,600 units. Max 6 uploads/day.
3. **Platform metadata generated by WF_PLATFORM_METADATA** — 3-5K description, SEO tags, platform-specific captions.
4. **If auto-pilot ON and all 13 QA checks pass:** auto-publish as UNLISTED (skip Gate 3).
5. **Write to Supabase after publish:** `topics.youtube_url`, `youtube_video_id`, `published_at`.
6. **Post-publish chain:** WF_VIDEO_METADATA stores SEO doc to Drive. Daily analytics cron begins.
7. **All API calls use WF_RETRY_WRAPPER.**

## Error Handling
- QA check failure: non-blocking for Gate 3 review. User can approve despite failed checks.
- YouTube upload failure: retry 3x. If quota exceeded, queue for next day.
- Metadata generation failure: use fallback template (title + basic description). Flag for manual edit.
- Thumbnail upload failure: retry. Video publishes without custom thumbnail if persistent.

## Post-Publish
- `WF_ANALYTICS_CRON` (daily): views, watch hours, CTR, revenue, subscribers
- `WF_PPS_CALIBRATE` (monthly): predicted vs actual PPS accuracy tuning
- `WF_REVENUE_ATTRIBUTION` (monthly): production cost to actual revenue per topic
